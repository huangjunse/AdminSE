    关于菜单

    1. 菜单接收的是 json 数据，格式为：

    [
      {
        "typename": "treeview",   /*菜单类型：'treeview'/null*/
        "url": "DashBoard.html",  /*url，可以是绝对或者相对*/
        "name": "DashBoard",      /*菜单名，后面可以采用 highLightMenu 来高亮该菜单*/
        "namecn": "仪表盘",       /*显示的中文*/
        "idpath": "1",            /*根据id产生的路径，形如 1>2>3*/
        "pid": null,              /*父id*/
        "id": 1,                  /*id*/
        "icon": "dashboard"       /*fontawesome图标，不用添加fa前缀*/
      }
    ]


    2. 产生 Menus 的方法：getMenuStr()

    可以直接添加到 menu 的容器中：

    $(".sidebar-menu").html(getMenuStr());


    3. 高亮菜单： highLightMenu('menuname');

    例如：

    highLightMenu('Readme');

    关于Django中的应用

    由于此模板是在 Django 项目中衍生出来的，因此，几乎所有内容都可以直接套用到 Django 中。

    使用的时候需要修改所有文件引用需要使用绝对地址。

    例如，默认情况下：
    1. Django 项目静态文件（ js/css/img ）存放在 app 的 static 文件夹下。
    2. 静态文件的访问 url 应该形如 http://localhost/static/.../js/base.js 。
    3. html 文件一般存放在 templates 文件夹下，但是页面的访问依赖于 view 的设置，因此，其实存放在哪里都没有关系。
    基于以上几点的页面，css的引用方式为：

    <link href="/static/dist/css/base.css" rel="stylesheet" type="text/css" />

    建议所有页面均采用 Django 模板的方式引入 头部、尾部 的公共部分

    {% include 'Template_File.html' %}
