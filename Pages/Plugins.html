    fastclick

    官网：fastclick-github

    该插件用于处理移动端 click 事件 300 毫秒延迟。

    1. 为什么存在延迟？

    根据 Google 开发者文档：
    ...mobile browsers will wait approximately 300ms from the time that you tap the button to fire the click event. The reason for this is that the browser is waiting to see if you are actually performing a double tap.
    从点击屏幕上的元素到触发元素的 click 事件，移动浏览器会有大约 300 毫秒的等待时间。为什么这么设计呢？ 因为它想看看你是不是要进行双击（double tap）操作。


    2. 兼容性

    • Mobile Safari on iOS 3 and upwards
    • Chrome on iOS 5 and upwards
    • Chrome on Android (ICS)
    • Opera Mobile 11.5 and upwards
    • Android Browser since Android 2
    • PlayBook OS 1 and upwards

    3. 不应用 FastClick 的场景

    1. 桌面浏览器；
    2. 如果 viewport meta 标签 中设置了 width=device-width， Android 上的 Chrome 32+ 会禁用 300ms 延时；
    3. viewport meta 标签如果设置了 user-scalable=no，Android 上的 Chrome（所有版本）都会禁用 300ms 延迟。
    4. IE10 中，可以使用 css 属性 -ms-touch-action: none 禁止元素双击缩放

    4. 使用：（建议加入全局）

    FastClick.attach(document.body);

    iCheck

    官网：iCheck-github

    中文：iCheck-中文

    CheckBox 和 Radio 的美化插件，特色：

    • 在不同浏览器和设备上都有相同的表现 — 包括 桌面和移动设备
    • 支持触摸设备 — iOS、Android、BlackBerry、Windows Phone
    • 支持键盘导航 — Tab、Spacebar、Arrow up/down 和其它快捷键
    • 方便定制 — 用HTML 和 CSS 即可为其设置样式 (试试 6 套针对Retina屏幕的皮肤吧)
    • 支持 jQuery 和 Zepto JavaScript工具库
    • 体积小巧 — gzip压缩后只有1 kb
    • 25 种参数 用来定制复选框（checkbox）和单选按钮（radio button）
    • 8 个回调事件 用来监听输入框的状态
    • 7 个方法 用来通过编程方式控制输入框的状态
    • 能够将输入框的状态变化同步回原始输入框中， 支持所有选择器

    示例：flat皮肤，blue色彩








    1. 使用：
    1.1 css及js：

    <link href="static/plugins/iCheck/all.css" rel="stylesheet" type="text/css" />
    <script src='static/plugins/iCheck/icheck.min.js'></script>
    1.2 Dom：
    <input type='checkbox'>
    1.3 激活：
    $("[type='checkbox']").iCheck();


    2. 参数：（参数太多，抽常用的来说）

    /* 复选框皮肤样式，'icheckbox_*_#'
       *可选为： minimal/square/flat/line/polaris/futurico
       #可选为： blue/purple/green/red/yellow/orange/aero/grey/pink*/
    checkboxClass: 'icheckbox',
    radioClass: 'iradio',    /* 单元框皮肤样式，参考checkboxClass*/

    3. 方法：
    $('input').iCheck('check'); — 将输入框的状态设置为checked
    $('input').iCheck('uncheck'); — 移除 checked 状态
    $('input').iCheck('toggle'); — toggle checked state
    $('input').iCheck('disable'); — 将输入框的状态设置为 disabled
    $('input').iCheck('enable'); — 移除 disabled 状态
    $('input').iCheck('update'); — apply input changes, which were done outside the plugin
    $('input').iCheck('destroy'); — 移除iCheck样式


    4. 事件：
    ifClicked 用户点击了自定义的输入框或与其相关联的label
    ifChanged 输入框的 checked 或 disabled 状态改变了
    ifChecked 输入框的状态变为 checked
    ifUnchecked checked 状态被移除
    ifDisabled 输入框状态变为 disabled
    ifEnabled disabled 状态被移除
    ifCreated 输入框被应用了iCheck样式
    ifDestroyed iCheck样式被移除

    placeholder

    官网：placeholder-github

    中文：placeholder-中文

    placeholder是一个Javascript库，用于在浏览器端生成placeholder image，提供完善的配置和简单的方法。

    • 不依赖任何 js 框架，比如 jq
    • 小于 1k ，具体是0.98k，增加img配置方式之后，为1.52kb
    • 配置项多，都有默认值
    • 使用简单

    示例1. 获取图片，如果获取失败则显示 Image 404：

    示例2. 自定义占位图片大小，并显示自定义文字。：


    1. 使用：
    1.1 js：

    <script src='static/plugins/placeholder/placeholder.min.js'></script>
    1.2 Dom1：调用placeholder.js的方法进行替换（注意，这种方式需要js文件放在调用之前）
    <img src="image_origin.png" onerror="this.src=placeholder.getData({text: 'Image 404'})"/>
    1.3 Dom2：使用URL参数的方式配置Img的属性并携带参数options
    <img options="size=256x128&text=Hello!" class="placeholder" />
    1.4 如果采用js方式产生
    $('imgselector')[0].src = placeholder.getData(options);


    2. 参数：（用在options属性中或者在placeholder.getData方法中均可）
    要注意的是：
    2.1 如果使用getData方法，参数的配置均需要添加引号，如size:'256x64'
    2.2 参数中的text值应该被encodeURIComponent转码，主要包括： % 为 %25, & 为 %26, = 为 %3D, 空格 为 %20 etc.

    • size: placeholder图片尺寸. 例如: 256x128, 默认: 128x128.
    • bgcolor: 背景颜色. 例如: #969696. 默认: random.
    • color: 前景颜色，文字颜色. 例如: #ccc. 默认: random.
    • text: 自定义文本内容. 例如: Hello World, 你好. 默认: equal to size.
    • fstyle: 字体样式. 可以是 normal / italic / oblique. 默认: oblique.
    • fweight: 字体weight. 可以是 normal / bold / bolder / lighter / Number. 默认: bold.
    • fsize: 字体大小. 默认: 自动 计算字体大小防止文字超出图片大小.
    • ffamily: 字体. 默认: consolas.

    3. 方法：
    placeholder.getData(opts): 获得placeholder图片的base64字符串，可以直接在img标签的src属性中使用，或者在css的背景中使用。
    placeholder.getCanvas(opts): 获得canvas元素, 可以直接插入到DOM结构中。


    4. 事件：
    暂缺--无。

    vivus

    官网：vivus

    vivus是一个不依赖于任何脚本的js库，能描画任何svg路径，缺点是这个路径需要用户自己画。


    示例
    类型：

    路径时间参数：

    动画时间参数：

    1. 使用：
    1.1 js：

    <script src='static/plugins/vivus/vivus.min.js'></script>
    1.2 Dom1+SVG：
    <img src="image_origin.png" onerror="this.src=placeholder.getData({text: 'Image 404'})"/>
    1.3 Dom2：使用URL参数的方式配置Img的属性并携带参数options
    <img options="size=256x128&text=Hello!" class="placeholder" />
    1.4 如果采用js方式产生
    $('imgselector')[0].src = placeholder.getData(options);


    2. 参数：（挑几个来翻译）

    参数默认值描述
    type delayed 动画类型：delayed, async, oneByOne, script, scenario or scenario-sync.
    file null Link to the SVG to animate. If set, Vivus will create an object tag and append it to the DOM element given to the constructor. Be careful, use the onReady callback before playing with the Vivus instance.
    start inViewport Defines how to trigger the animation (inViewport once the SVG is in the viewport, manual gives you the freedom to call draw method to start, autostart makes it start right now)
    duration 200 动画持续的时间，单位：帧
    delay integer 画第一条和最后一条路径的时间延迟，单位：帧 (仅delayed模式下有效).
    onReady function Function called when the instance is ready to play.
    pathTimingFunction function 每一条路径的时间函数。赋值方式类似于： Vivus.LINEAR/Vivus['LINEAR']
    animTimingFunction function SVG整体的时间函数。赋值方式参考 pathTimingFunction
    dashGap integer Whitespace extra margin between dashes. Increase it in case of glitches at the initial state of the animation. [Default: 2]
    forceRender boolean Force the browser to re-render all updated path items. By default, the value is true on IE only. (check the 'troubleshoot' section for more details).
    selfDestroy boolean Removes all extra styling on the SVG, and leaves it as original.

    3. 方法：

    方法描述
    play(speed) 图画的显示速度，0~1表示慢速，大于1快速。[默认: 1]
    stop() 停止画图。
    reset() Reinitialises the SVG to the original state: undrawn.
    finish() Set the SVG to the final state: drawn.
    setFrameProgress(progress) Set the progress of the animation. Progress must be a number between 0 and 1.
    getStatus() Get the status of the animation between start, progress, end
    destroy() Reset the SVG but make the instance out of order.

    暂缺--无。
